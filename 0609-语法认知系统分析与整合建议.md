## 一、0609 文档的核心创新：你做了什么

阅读 0609 文档后，第一感受是：**这不是一份语法讲义，这是一套"语法认知操作系统"**。

传统语法教学的问题本质上是**认知模型错位**：

```
传统路径：
规则名称（如"现在完成时"）→ 公式（have/has + done）→ 使用条件（4条规则）→ 练习
    ❌ 学生在大脑里存了一堆 if-then 规则，写作时来不及调用
    ❌ 规则之间相互孤立，没有统一的认知框架
    ❌ "为什么这里用这个时态？"——学生只能回答"因为规则说..."
```

你做的事情本质上是**用视觉隐喻替换规则记忆**：

```
0609 路径：
视觉隐喻（📸 / 🎬 / 📂 / 📦）→ 认知画面（"你在看什么？"）→ 直觉判断 → 时态选择
    ✅ 一张图对应一种时态的本质含义
    ✅ 所有时态共享同一套"注意力模型"（你现在盯着哪里看？）
    ✅ "为什么用这个时态？"——学生回答"因为我现在看的是照片/档案/录像带"
```

### 你创造的认知隐喻体系

| 时态/结构 | 视觉隐喻 | 认知本质 | 一句话直觉 |
|----------|---------|---------|-----------|
| Past Simple | 📸 旧照片 | 事件已封存、已完结 | "墙上挂着一张历史照片" |
| Past Continuous | 🎬 录像带切片 | 进入时间内部观察过程 | "我正在播放那段录像" |
| Present Perfect | 📂 档案记录 | 站在现在盘点过去 | "我手里攥着这份经历档案" |
| Present Perfect Continuous | 📂🎥 档案+持续录制 | 从过去持续到现在未停 | "这件事已经做了一阵子，做到现在" |
| Used to | 📦 封印档案盒 | 过去状态与现在彻底断裂 | "那个旧版本的我已被封箱" |
| be used to | 📷 静态照片 | 当前的状态属性 | "现在的我已经适应了" |
| Simple vs Continuous 对比 | 📷前景点 vs 🎬背景舞台 | 叙事功能分层 | "哪个推剧情？哪个搭舞台？" |

### 这套系统的三个突破性贡献

**贡献一：统一了时态教学的"解释语言"**

所有时态不再各自为政。学生只需问一个问题："我的注意力现在落在哪里？"——同一个问题，同一个认知模型，覆盖全部时态选择。

**贡献二：破解了"中文母语负迁移"的根源问题**

中文没有时态屈折变化，所以中国学生天然缺乏"时态意识"。传统教学试图通过规则灌输来建立这个意识，但规则是外挂的。你的隐喻系统直接跳过了"规则翻译"这一步——📸 和 📂 是非语言的视觉直觉，不分中英文。学生在写英语句子之前，先在脑内"看到了"照片还是档案，然后自然选择对应时态。

**贡献三：为 AI 辅助写作提供了可操作的"语法诊断语言"**

当前的 topicData.js 中，highlight 的 type 只有 `connector`, `adverb`, `non-finite`, `non-restrictive`, `concrete-imagery`, `concrete-feeling`, `specific-detail`, `change-verb`, `mind-verb`, `object-clause`。这些都是**修辞/风格维度**的标注。0609 文档提供了**语法认知维度**的第二套标注体系，可以直接嵌入现有的 A/B 发现句对比中。

## 二、与现有系统的对接点分析

### 2.1 现有系统的三层架构

从系统框架文档和代码中可以看到一个清晰的三阶架构：

```
第一阶：审题破题力
    └── 拿到题目 → 3问破题 → 骨架速填 → 知道"写什么"

第二阶：意义表达力
    └── 14种功能表达 → 每句话"承担什么战术任务" → 知道"写什么功能的句子"

第三阶：逻辑衔接力
    └── 衔接三问 + 四种驱动力 → 句与句之间"如何召唤下一句"
```

在这个架构中，**语法知识目前是隐性的**。它散落在：
- topicData.js 的 `grammar` 字段（一行文本描述）
- sentence patterns（句型模板）
- highlight labels（句子标注中偶尔提及语法结构）

### 2.2 语法在现有系统中缺失了什么

看一下现有代码中语法的处理方式：

```javascript
// topicData.js 中的 grammar 字段
grammar: "一般现在时 + 形容词表语；because / for 引出原因"
```

这行描述的问题是：
1. **它是元语言描述**（"一般现在时"、"形容词表语"），不是认知直觉
2. **它没有解释"为什么"**——为什么这里用一般现在时而不是别的？
3. **它不与学生的写作决策关联**——学生看到这行字，不会因此做出更好的时态选择

而你的 0609 文档给出的等价描述应该是：

> 📸 **认知画面**：你在描述一个**不受时间限制的普遍事实**（"春天是我最爱的季节"——这不是过去某个时刻的事，而是一个跨越时间的个人真理）。所以用一般现在时，就像在说"地球围绕太阳转"一样——它是照片，不是录像带，不是档案。

## 三、核心建议：将语法认知系统化

### 3.1 建立与词汇认知平行的"语法认知维度"

你现有的词汇认知系统有一个明确的三层结构（在 topicData.js 中可见）：

```
词汇认知三层：
  concrete-imagery   → 具象画面（看得见、摸得着的东西）
  concrete-feeling   → 具象感受（可感知的情绪状态）
  specific-detail     → 具体细节（时间、地点、方式）
```

**建议：为语法建立平行的三层认知维度**：

```
语法认知三层：
  📸 snapshot-mode    → 静态事实（一般现在时、一般过去时、静态动词）
  🎬 process-mode     → 过程展开（进行时、非谓语伴随状语）
  📂 archive-mode     → 档案盘点（完成时、used to、情态动词完成式）
```

这三个认知维度与你的 5 种推理链型天然对应：

| 链型 | 主导语法认知模式 | 原因 |
|------|----------------|------|
| 论证链 (argument) | 📸 snapshot + 📂 archive | 表达观点是静态判断；总结认识是档案盘点 |
| 画面链 (imagery) | 📸 snapshot + 🎬 process | 静态画面 + 动态元素在画面中展开 |
| 因果链 (causality) | 📸 snapshot（事件）+ 🎬 process（过程中） | 动作结果是照片；过程细节是录像带 |
| 对比链 (contrast) | 📸 vs 📸 或 📂 vs 📦 | 两张照片对比；档案 vs 封箱 |
| 时间层链 (time-layer) | 📂 archive | 回顾过去，站在现在盘点 |

### 3.2 具体实施方案

#### 方案 A：增强现有 topicData.js 中的 grammar 字段

当前：
```javascript
grammar: "一般现在时 + 形容词表语；because / for 引出原因"
```

改为：
```javascript
grammar: {
  tenseHint: "📸 snapshot-mode",
  cognitiveExplanation: "你在陈述一个不受时间限制的个人真理（你最喜欢的季节），就像说'太阳从东边升起'一样——这不是某个特定时刻的事件，而是跨越时间的静态事实。所以用一般现在时。",
  studentQuestion: "这句话是在描述一个永久事实，还是在讲述一个特定时刻发生的事？",
  commonError: "❌ I am liking spring very much. → 状态动词 like 不能用进行时，因为它本身已经是 📸 静态画面，不需要加 🎬。"
}
```

#### 方案 B：在 discovery.highlights 中新增 tense-choice 类型

当前 highlight 类型：
```
connector, adverb, non-finite, non-restrictive, concrete-imagery,
concrete-feeling, specific-detail, change-verb, mind-verb, object-clause
```

建议新增：
```javascript
// 新增 highlight type
{ type: "tense-choice", label: "📸 一般现在时 — 这是一个超越时间的个人真理，不是某个时刻的事件" }
{ type: "tense-choice", label: "📂 现在完成时 — 说话人在盘点'到目前为止'的人生档案" }
{ type: "tense-choice", label: "🎬 过去进行时 — 正在播放一段背景录像，为即将发生的 📸 事件搭建舞台" }
{ type: "tense-choice", label: "📦 used to — 过去的循环已被封箱，与现在彻底断裂" }
```

#### 方案 C：新增"语法认知追问"到 reasoningChains.js

在每个推理链的追问序列中插入一个**时态决策追问**：

```javascript
// 插入到 reasoningChains.js 的每个 chain.steps 中
{
  id: "tenseDecision",
  label: "时态决策：照片、录像带还是档案？",
  question: (topicTitle, sentRole, prevAnswers) => {
    return `你现在要写的这句话——在你的大脑里，它更像一张已经拍好的 📸 照片、一段正在播放的 🎬 录像带、还是一份你正在翻看的 📂 档案记录？为什么？`;
  },
  hint: "📸 照片 = 事件已经结束，结果是确定的 | 🎬 录像带 = 你在描述过程本身，让它'播放'出来 | 📂 档案 = 你站在现在，盘点过去到现在累积的东西",
  answerType: "choice",
  options: ["📸 照片（一般现在/一般过去）", "🎬 录像带（进行时/非谓语伴随）", "📂 档案（完成时/used to）"],
  lang: "zh",
}
```

## 四、语法认知与词汇认知的统一框架

### 4.1 统一认知模型

最终目标是将语法认知和词汇认知统一在**同一个"大脑画面"模型中**：

```
                    ┌─────────────────────────────┐
                    │   学生脑中的"认知画面"       │
                    │                             │
                    │  "我看到/想到什么？"          │
                    └─────────────┬───────────────┘
                                  │
            ┌─────────────────────┼─────────────────────┐
            │                     │                     │
    ┌───────▼───────┐    ┌───────▼───────┐    ┌───────▼───────┐
    │  词汇选择      │    │  语法选择      │    │  句型选择      │
    │               │    │               │    │               │
    │ concrete-     │    │ 📸 snapshot   │    │ simple /      │
    │ imagery       │    │ 🎬 process    │    │ compound /    │
    │ concrete-     │    │ 📂 archive    │    │ complex       │
    │ feeling       │    │               │    │               │
    │ specific-     │    │               │    │               │
    │ detail        │    │               │    │               │
    └───────────────┘    └───────────────┘    └───────────────┘
            │                     │                     │
            └─────────────────────┼─────────────────────┘
                                  │
                    ┌─────────────▼───────────────┐
                    │   输出：一句有画面、有时态、  │
                    │   有结构的英语句子            │
                    └─────────────────────────────┘
```

### 4.2 一个具体的统一示例

以 Topic 1, Sentence 8（具体回忆）的 A 版句为例：

> *"I still remember clearly one spring afternoon when I discovered a hidden garden full of butterflies, and I stood there silently, completely lost in its magical beauty."*

**统一认知标注**（三层并行）：

```
📸📸📂 时态认知层：
  "I still remember"     → 📂 现在时 + still = 档案中的持久记忆
  "I discovered"         → 📸 过去时 = 那个下午是封存的旧照片
  "I stood there"        → 📸 过去时 = 照片中的定格画面
  "completely lost"      → 📸 过去分词作状语 = 照片中人物的状态

🎨 词汇认知层：
  "hidden garden"        → concrete-imagery（具象地点）
  "full of butterflies"  → concrete-imagery（具象画面）
  "magical beauty"       → concrete-feeling（具象感受）
  "silently"             → adverb（方式感）
  "clearly"              → adverb（记忆清晰度）

🔗 句型结构层：
  "when I discovered..." → 定语从句（时间锚定）
  "completely lost..."   → 非谓语伴随状语（状态补充）
```

## 六、一个非显而易见的建议

审视 0609 文档和现有系统的关系，我发现了一个你很可能已经直觉到的模式：

**0609 文档中最好的内容不是时态规则解释，而是那套"自问自答系统"**。

就是这段：

> 当学生准备写一个发生在过去的动作时，不要问时间，而是问自己：
> **"我的注意力，现在正盯着哪里看？"**
>
> 视角一：注意力指向"过去的墙壁" 📸 → Past Simple
> 视角二：注意力指向"现在的双手" 📂 → Present Perfect

这与你现有的 **reasoningChains.js 追问系统**完全同构！

reasoningChains.js 的核心设计就是：**不教知识，而是教学生"问自己什么问题"**。你在 0609 文档中对时态做的事情，本质上是**把时态选择变成了一个追问**，而这个追问恰好可以无缝嵌入现有的追问链。

所以，语法认知系统化最优雅的方式不是"加一套语法课程"，而是：

> **把语法选择转化为"追问"，嵌入到每条推理链的追问序列中。**

这样，学生在进行论证链、画面链、因果链的思考时，时态决策会作为一个**自然的思考步骤**出现，而不是事后检查的"语法纠错"。
